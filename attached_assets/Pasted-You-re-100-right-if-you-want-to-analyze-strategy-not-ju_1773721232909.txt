def absorption_score(trades):

    volume = sum(t["size"] for t in trades)

    price_move = max(t["price"] for t in trades) - min(t["price"] for t in trades)

    if price_move == 0:
        return volume

    return volume / price_move

Large values indicate stealth accumulation.

3. Order-Flow Imbalance (Predicting Moves)

This is the most powerful quantitative signal.

Order flow measures buy pressure vs sell pressure.

Formula:

OFI = buy_volume − sell_volume

Compute over a rolling window.

Example:

OFI = +50k

Positive imbalance predicts price increase.

Normalized Imbalance

To compare across markets:

imbalance_ratio = (buy − sell) / (buy + sell)

Range:

-1 → strong sell pressure
0 → neutral
+1 → strong buy pressure
Implementation

Example Python:

def order_flow_imbalance(trades):

    buy = sum(t["size"] for t in trades if t["side"] == "BUY")
    sell = sum(t["size"] for t in trades if t["side"] == "SELL")

    total = buy + sell

    if total == 0:
        return 0

    return (buy - sell) / total

Below is a complete quantitative trader-scoring framework for prediction markets. The goal is to rank every wallet by true skill, not just profit. This is similar to how hedge funds evaluate traders internally.

We’ll build a multi-factor scoring system that measures:

information advantage

timing skill

1. Why PnL Alone Is a Bad Metric

Raw PnL is misleading because it depends heavily on:

wallet size
market volatility
lucky outcomes

Example:

Trader A: +$500k
Trader B: +$80k

But:

Trader A volume = $50M
Trader B volume = $800k

Trader B is far more skilled.

So the system must normalize performance.

2. Core Metrics for the Quant Score

We calculate six main factors.

1. Closing Line Value (CLV)
2. Entry Edge
3. Market Impact
4. Risk-Adjusted Returns
5. Trade Timing
6. Consistency

Each metric is normalized and combined.

3. Closing Line Value (CLV)

CLV measures whether a trader enters at better probabilities than the final market consensus.

Formula:

CLV = close_price − entry_price

Example:

entry: 0.42
close: 0.55
CLV = +0.13

Positive CLV means the trader beat the market.

Wallet CLV

Average across trades:

wallet_clv = mean(CLVi)

Normalized:

CLV_score = min(max(wallet_clv / 0.10, 0), 1)

Interpretation:

0.00 → poor
0.05 → good
0.10+ → elite
4. Entry Edge

Entry edge measures short-term predictive power.

Definition:

entry_edge = price_after_30m − entry_price

Example:

entry: .41
30m later: .46
edge = +5%

This captures informational advantage.

Wallet Entry Edge
entry_edge_wallet = mean(edge_i)

Normalize:

edge_score = clamp(entry_edge_wallet / 0.05)

Interpretation:

>3% average → strong signal trader
5. Market Impact

Some traders move the market.

Impact metric:

impact = price_after_5m − price_before_trade

Wallet impact score:

impact_score = mean(abs(impact_i))

But we also measure direction correctness.

Directional impact:

correct_impact = sign(trade_direction) * impact

Positive means the trader moved price in their favor.

6. Risk-Adjusted Return

We measure Sharpe-like performance.

Formula:

Sharpe = mean(PnL_per_trade) / std(PnL_per_trade)

Or use ROI:

ROI = profit / total_cost

Risk score:

risk_score = ROI / volatility

High score means consistent profitable trades.

7. Trade Timing Score

Detect traders who react earliest to events.

Metric:

timing = mean(price_2m_after − entry_price)

Example:

entry .42
2m later .45
timing edge = +3%

Normalize:

timing_score = clamp(timing / 0.03)
8. Consistency Score

Many traders are profitable only during one lucky event.

We measure stability across time.

Steps:

divide history into monthly buckets

compute monthly ROI

compute standard deviation

Consistency formula:

consistency = mean(monthly_ROI) / std(monthly_ROI)

High score = stable performance.

9. Market Selection Skill

Good traders choose markets with high edge.

Metric:

avg_market_volatility

profit_per_market

Compute:

market_skill = pnl / number_of_markets

Professional traders scale positions based on confidence.

Measure correlation:

position_size vs edge

Formula:

sizing_skill = corr(size, CLV)

Positive correlation means:

larger bets when edge is higher

3. Predictive Model Types

Several approaches:

a. Simple Statistical Model

Weighted linear regression:

Δfuture_price ~ β1 * orderflow + β2 * sharp_signal + β3 * depth + β4 * trend

Output: expected price change (ΔP) → implied probability.

b. Logistic Regression / Classification

Binary outcome: probability goes up or down in next window.

logit(P(up)) = w1*orderflow + w2*sharp_buy - w3*sharp_sell + ...

Pros: interpretable

Cons: less flexible on nonlinear patterns

c. Gradient Boosted Trees (XGBoost / LightGBM)

Features: all above engineered metrics

Label: Δprice / Δprobability over next 5–30 minutes

Handles nonlinear interactions (e.g., sharp traders + low liquidity)

d. Deep Learning / LSTM

Input: sequence of trades & features over time

Captures temporal dependencies and momentum

Output: predicted probability shift

4. Training Targets

Define the target variable:

Δprob = future_midprice − current_midprice

or

Δprob_class = {Up, Down, Neutral}

Use rolling windows to create training samples:

Window size: 5–15 min
