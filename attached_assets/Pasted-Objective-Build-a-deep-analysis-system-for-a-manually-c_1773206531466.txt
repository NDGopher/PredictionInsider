Objective
Build a deep analysis system for a manually curated list of elite traders. For each trader: pull full trade history (paginated until complete), compute deep analytics (ROI by sport/market type, bet sizing consistency, account age, trades/day, specialization tags, Sharpe-like consistency score), store results in PostgreSQL, display in a new Elite Profiles page, and run periodic re-analysis every 24h. Integrate curated trader presence into signal scoring.

T002: Build trade history fetcher + analysis engine (backend)
Blocked By: [T001]
Details:
Create server/eliteAnalysis.ts with:
fetchFullTradeHistory(wallet): paginate DATA_API /trades?user={wallet}&limit=1000&offset=N until empty; join each trade with Gamma API for sport/market-type classification; store raw trades in eliteTraderTrades table
computeTraderProfile(wallet): reads stored trades, computes ALL metrics below, saves to eliteTraderProfiles
Metrics to compute:
Volume: totalUSDC, avgBetSize, medianBetSize, betSizeStdDev, betSizeConsistencyScore (1 - stdDev/avg, capped 0-1)
ROI & PNL: overallROI, overallPNL, last30dROI, last90dROI, winRate, totalTrades
PNL consistency: rollingROIByMonth (array), sharpeScore (avg monthly ROI / std dev), maxDrawdownStreak (consecutive losing months)
Sport specialization: roiByTag (NBA, NFL, NHL, MLB, Soccer, UFC, Tennis, eSports, Politics, Other) + tradeCountByTag — only include tags with ≥5 trades
Market type specialization: roiByMarketType (moneyline, spread, total, futures, other) + tradeCountByMarketType
Activity: firstTradeDate, daysSinceFirst, tradesPerDay (total / daysSinceFirst), avgTradesPerWeek
Bet sizing relative to self: identify top 10% largest bets → "big bet ROI" vs "small bet ROI" (does size predict conviction?)
Tags: auto-assign specialization tags e.g. "NBA Expert", "O/U Specialist", "Soccer Pro", "Futures Whale", "Consistent Grinder" based on top ROI categories with ≥10 trades
qualityScore: weighted composite (overallROI×0.25 + sharpeScore×0.25 + last90dROI×0.15 + winRate×0.10 + betSizeConsistency×0.10 + tradeCount bonus×0.15)
Acceptance: calling computeTraderProfile on a real wallet returns a populated profile object with all fields
